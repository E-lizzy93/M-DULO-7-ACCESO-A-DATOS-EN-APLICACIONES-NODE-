Estructura del Proyecto:

/db: Módulo que gestiona la conexión a PostgreSQL y contiene la lógica de transacciones SQL (BEGIN, COMMIT, ROLLBACK) para asegurar la integridad de las órdenes.

/public: Interfaz de usuario donde se realizan las simulaciones de compra, validación de inventario y generación de despachos.

/scripts: Contiene los archivos SQL necesarios para la creación de tablas de productos, órdenes y stock inicial.

server.js: Servidor Express que procesa las peticiones de compra, coordinando las validaciones previas antes de confirmar cualquier transacción en la base de datos.

package.json: Registro de las dependencias del proyecto, incluyendo el driver pg para la comunicación con PostgreSQL.
